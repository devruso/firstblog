REGULAR EXPRESSIONS

The .test()str method takes the regex, applies it to a string (which is placed inside the parentheses), and returns true or false if your pattern finds something or not.
You can also extract the actual matches you found with the .match() method.
To search or extract a pattern more than once, you can use the g flag.
Sometimes you won't (or don't need to) know the exact characters in your patterns.The wildcard character . will match any one character. ex: /.un/ returns: sun,run,fun,bun etc.
You can search for multiple patterns using the alternation or OR operator: |. ex:/yes|no|maybe/ ;
But sometimes, you might want to also match case differences, in this situation you can use the i flag. ex: /ignorecase/i;
You can search for a literal pattern with some flexibility with character classes. Character classes allow you to define a group of characters you wish to match by placing them inside square ([ and ]) brackets. ex: /b[aiu]g/;
Inside a character set, you can define a range of characters to match using a hyphen character: -. ex: /[a-e]at/;
Using the hyphen (-) to match a range of characters is not limited to letters. It also works to match a range of numbers. ex: let myRegex = /[a-z0-9]/ig;
To create a negated character set, you place a caret character (^) after the opening bracket and before the characters you do not want to match. ex: /[^aeiou]/gi
You can use the + character to check if that is the case. Remember, the character or pattern has to be present consecutively. That is, the character has to repeat one after the other. ex: /a+/g;
There's also an option that matches characters that occur zero or more times. ex: let goRegex = /go*/;.
In regular expressions, a greedy match finds the longest possible part of a string that fits the regex pattern and returns it as a match. The alternative is called a lazy match, which finds the smallest possible part of the string that satisfies the regex pattern. ex: t[a-z]*?i/ returns ["ti"].


Outside of a character set, the caret is used to search for patterns at the beginning of strings. ex: let calRegex = /^Cal/; You can search the end of strings using the dollar sign character $ at the end of the regex. ex: let storyRegex = /story$/; To look for one or more characters:+ .
To match for zero or more characters: *.
If you want to match a certain range of patterns with quantity specifiers (Quantity specifiers are used with curly brackets ({ and }). Example: /Oh{3,6}\sno/;
Sometimes you only want to specify the lower number of patterns with no upper limit: /haz{4,}ah/ ;
You can specify the lower and upper number of patterns with quantity specifiers using curly brackets:/tim{4}ber/ ;
Sometimes the patterns you want to search for may have parts of it that may or may not exist: /favou?rite/;
How to match white space: \s .
How to match everything but white space: \S.
The closest character class in JavaScript to match the alphabet is \w. This shortcut is equal to [A-Za-z0-9_]. This character class matches upper and lowercase letters plus numbers. ex: /[\w+]/g
You can search for the opposite of the \w with \W. Note, the opposite pattern uses a capital letter. This shortcut is the same as [^A-Za-z0-9_].
The shortcut to look for digit characters is \d, with a lowercase d. This is equal to the character class [0-9] , which looks for a single character of any number between zero and nine.
The shortcut to look for non-digit characters is \D . This is equal to the character class [^0-9] , which looks for a single character that is not a number between zero and nine.
Sometimes we want to check for groups of characters using a Regular Expression and to achieve that we use parentheses ().If you want to find either Penguin or Pumpkin in a string, you can use the following Regular Expression: /P(engu|umpk)in/g. /(Franklin|Eleanor) .*Roosevelt/g;
Regular Expressions: Reuse Patterns Using Capture Groups. To specify where that repeat string will appear, you use a backslash (\) and then a number. This number starts at 1 and increases with each additional capture group you use. An example would be \1 to match the first group.
Searching is useful. However, you can make searching even more powerful when it also changes (or replaces) the text you match.You can search and replace text in a string using .replace() on a string. ex: "Code Camp".replace(/(\w+)\s(\w+)/, '$2 $1');


Lookaheads are patterns that tell JavaScript to look-ahead in your string to check for patterns further along. A positive lookahead will look to make sure the element in the search pattern is there, but won't actually match it. A positive lookahead is used as (?=...) where the ... is the required part that is not matched. On the other hand, a negative lookahead will look to make sure the element in the search pattern is not there (?!...).


test example: Use lookaheads in the pwRegex to match passwords that are greater than 5 characters long, do not begin with numbers, and have two consecutive digits.
answer: let pwRegex = /^\D(?=\w{5,})(?=\w*\d{2})/;
